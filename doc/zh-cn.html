用户守则

Cloud I Wanna 用户须遵守以下规则：
- 禁止任何方式的游戏作弊，包括但不限于代码修改、变速工具、按键脚本、一键JC
- 禁止恶意刷分、恶意举报
- 禁止涉及违规的言论、恶意言论、人身攻击
- 禁止发布无意义、无解的游戏，游戏内禁止出现违规内容
- 禁止任何方式的爬取、篡改数据库、修改应用程序
注：违规内容包括但不限于涉及政治、广告、引战、谣言、色情、血腥暴力、猎奇的内容

应用相关

应用缓存

Cloud I Wanna 使用本地缓存以提高加载速度和支持离线功能，具体缓存内容如下：

 - 基础缓存：代码缓存，字体缓存，应用数据等

 - 游戏缓存：游戏列表，游戏数据

 - 图像缓存：用户头像，游戏封面图，游戏内贴图、背景图片

 - 音频缓存：游戏内音效、背景音乐

 - 其它缓存：消息记录，玩家存档，制作项目

缓存的使用十分频繁，因此不建议经常清理。当缓存占用空间过大时，可在设置中清理应用缓存，此操作只会清理游戏缓存、图像缓存和音频缓存

离线使用

离线情况下，可使用的功能如下：

游戏：仅已缓存的游戏，且恢复联网后已有数据不会上传

制作：仅可使用已缓存的贴图、背景图片、音效，背景音乐不可用

注意：清理缓存将影响应用的离线使用

创作值

为了提高作品质量，Cloud I Wanna 使用创作值衡量作者的创作水平

创作值计算规则

创作值初始为 20，上限 100，下限 0，计算规则如下：
	游戏评分 > 4.5		+5
	游戏评分 > 4		+2
	游戏评分 > 3		+1
	游戏评分 < 2		-1
	游戏评分 < 1		-2
	游戏评分 < 0.5		-5
	恶意高难/无解		-10
	无意义/水			-10
	违规内容			-20

创作值作用

	创作值 = 100		可使用实验性功能
	创作值 < 10			需要通关才能发布游戏
	创作值 = 0			无法发布游戏

行为值

为了规范用户行为，Cloud I Wanna 使用行为值限制用户的行为

行为值计算规则

行为值初始为 100，上限 100，下限 0，增减规则如下：
	恶意刷分、恶意举报					-20
	违规言论、恶意言论、人身攻击			-20
	游戏出现违规内容						-30
	游戏作弊								-50
	爬取、篡改数据库、修改应用程序		-100
	行为值不足100							每日+1

行为值作用

	行为值 < 70		无法评分
	行为值 < 50		无法评论
	行为值 < 20		无法上传通关记录
	行为值 = 0		封禁，视严重程度决定封禁时长

引擎特性

Cloud I Wanna 游戏引擎使用原生 JavaScript 开发，完全还原常规 I Wanna 操作手感，同时修复了由 GameMaker 引擎引起的bug。

录像回放

游戏联机

联机功能

Cloud I Wanna 提供了方便的游戏联机功能

联机模式

普通模式：仅显示玩家

竞技模式：竞技将强制所有玩家同时从头开始，系统会推送玩家的游戏进度信息

合作模式：玩家间将共享游戏数据，包括存档、全局变量、机关、Boss血量等。合作模式下不会录像，通关记录也不会上传。

游戏性能

性能概述

Cloud I Wanna 游戏引擎使用原生 JavaScript 开发，非编译的特性使得其运行速度低于编译程序，因此在开发时做了大量优化工作以提高游戏性能：

 - 分层分区绘制，静态层和动态层使用不同的绘制频率，减少不必要的重绘

 - 使用多线程优化，不同的逻辑/绘制部分使用不同的线程以提高性能

 - 开启硬件加速 (部分设备上硬件加速可能会出现问题，可在设置中关闭)

虽然如此，由于Cloud I Wanna 制作游戏功能复杂，不正确的操作会极大地降低游戏性能。游戏运行速度和物体数量成反比，耗时最多的部分是碰撞检测和图像绘制，且旋转和缩放会进一步降低其运行速度，除此之外，不恰当的特效也会影响游戏性能。

性能优化

游戏中
 - 降低绘制帧率：如果游戏运行出现卡顿，可在设置中降低绘制帧率到30 (不会影响游戏运行的帧率)
 - 关闭不需要的特效
制作时
 - 避免大房间，使用多个小房间来代替，同时减少不必要的物体使用
 - 减少碰撞检测的使用，将碰撞逻辑放在物体数量少的一方，将没有碰撞逻辑的物体碰撞属性设为“无碰撞”
 - 设置条件触发的条件时，碰撞检测作为最后的条件 (如使用“[可见]且[碰撞固体]”而不是“[碰撞固体]且[可见]”)
 - 尽量使用固定视野而不是跟随视野
 - 尽量减少物体的旋转和缩放
 - 制作弹幕请使用经过优化的“创建弹幕”动作，而不是使用普通物体实现
